# แนวทางการตั้งค่า Default ของแต่ละ Process สำหรับเกษตรกรในแอป Sookta

วันที่จัดทำ: 2 มิถุนายน 2569

เอกสารนี้จัดทำเพื่อใช้เป็นข้อเสนอให้ทีมวิจัยพิจารณาค่าเริ่มต้นของข้อมูลเชิงงานในแอป Sookta สำหรับการประเมินความเสี่ยงด้านสรีรศาสตร์ของเกษตรกร โดยค่าเหล่านี้ควรถือเป็น “ค่าช่วยเริ่มต้น” ไม่ใช่ค่าจริงสุดท้ายของผู้ใช้งานแต่ละราย

## หลักการสำคัญ

1. ค่า default ไม่ควรเป็นตัวตัดสินผลลัพธ์สุดท้ายโดยลำพัง เพราะอาจทำให้คะแนนความเสี่ยงเกิด bias ได้
2. ท่าทางของร่างกายควรให้ภาพถ่ายหรือวิดีโอผ่าน MoveNet เป็นตัวคำนวณหลัก แล้วนำไปแปลงเป็นองค์ประกอบของ REBA
3. ค่าที่กล้องวัดไม่ได้ เช่น น้ำหนักที่ยก ระยะทาง ความถี่ ระยะเวลา และแรงดัน/ลาก ควรใช้ค่า default ตามประเภทงาน และเปิดให้แก้ไขหรือยืนยันได้
4. ข้อมูลที่ export สำหรับนักวิจัยควรมี field ระบุแหล่งที่มาของข้อมูล เช่น `default`, `auto_from_pose`, `user_confirmed`, หรือ `researcher_measured`
5. สำหรับเกษตรกร ไม่ควรให้กรอกตัวเลขจำนวนมาก ควรใช้ตัวเลือกง่าย เช่น เบา / ปานกลาง / หนัก แล้วให้ระบบแปลงเป็นค่าตัวเลขภายใน

## ค่า Default หลักที่แนะนำต่อ Process

| Process | วิธีประเมินหลัก | น้ำหนักตั้งต้น | ระยะเวลา | หมายเหตุ |
|---|---:|---:|---:|---|
| ปลูกกล้า | REBA + repetitive | 0 kg | 1 ชม. | เน้นก้ม ย่อ คุกเข่า ให้ posture จาก MoveNet คำนวณ |
| ใส่ปุ๋ย | REBA + ISO11228-1 | 10 kg | 1 ชม. | ใช้กับการยกหรือถือถัง/กระสอบปุ๋ย |
| พ่นสาร | REBA + ISO11228-2/3 | 15 kg ถ้าเป็นถังสะพาย | 1 ชม. | ถ้าเป็นเครื่องพ่นสะพาย ให้คิดน้ำหนักถังร่วมด้วย |
| ตัดแต่งกิ่ง | REBA + repetitive | 1 kg | 1 ชม. | เน้นแขนยกสูง ข้อมือซ้ำ และท่าทางจากภาพ |
| เก็บเกี่ยว | REBA + ISO11228-1/3 | 5 kg | 1 ชม. | ใช้กับตะกร้าหรือภาชนะเก็บผลผลิต |
| ขนย้ายผลผลิต | REBA + ISO11228-1/2 | 15 kg | 1 ชม. | ควรเปิด ISO11228 ร่วมกับ REBA |

## ค่า Default กลางของระบบ

| Field | ค่าแนะนำ | หมายเหตุ |
|---|---:|---|
| รายได้ต่อวัน | 300 บาท/วัน | ควรปรับตามพื้นที่หรือข้อมูลจากทีมวิจัย |
| horizontal distance | 25-35 cm | เลือกตาม process |

## แนวทางสำหรับค่า REBA

ค่า REBA posture ควรเริ่มจากค่า neutral ก่อน เช่น trunk, neck, leg, arm และ wrist เท่ากับ 1, twist เป็น false และ load/coupling/activity เป็น 0 แล้วให้ผลจาก MoveNet override ค่าเหล่านี้หลังประมวลผลภาพ

เหตุผลคือ หากตั้งค่า posture default ตาม process ตั้งแต่ต้น คะแนนอาจสูงหรือต่ำเกินจริงทั้งที่ยังไม่ได้ประเมินจากภาพจริง

## วิธีทำให้ใช้งานง่ายสำหรับเกษตรกร

หน้าจอของเกษตรกรควรถามเป็นภาษาง่าย และใช้ตัวเลือกแทนการกรอกตัวเลข เช่น

1. น้ำหนักที่ถือ: ไม่มี / เบา / ปานกลาง / หนัก
2. ระยะทาง: ไม่มี / ใกล้ / ไกล
3. ทำต่อเนื่อง: น้อยกว่า 30 นาที / ประมาณ 1 ชั่วโมง / มากกว่า 2 ชั่วโมง
4. ใช้อุปกรณ์เข็นหรือลาก: ใช่ / ไม่ใช่

ระบบสามารถ map ตัวเลือกเหล่านี้เป็นตัวเลขหลังบ้านเพื่อใช้คำนวณ REBA, ISO11228 และ economic impact ได้ โดยผู้ใช้งานไม่ต้องเข้าใจสูตรคำนวณ

## Checklist สำหรับทีมวิจัยก่อนนำไปใช้จริง

1. ยืนยันค่า default ของแต่ละ process ว่าสอดคล้องกับบริบทพื้นที่เก็บข้อมูล
2. ยืนยันรายได้ต่อวันหรือสูตร economic impact ที่ต้องใช้ในโครงการ
3. ยืนยันว่าค่าเพศ อายุ และ profile จะมาจากผู้ใช้หรือเจ้าหน้าที่วิจัย
4. ยืนยัน mapping ระหว่างตัวเลือกง่ายสำหรับเกษตรกรกับค่าตัวเลขหลังบ้าน
5. ตรวจสอบ export ว่ามีข้อมูลเพียงพอสำหรับวิเคราะห์ซ้ำและ audit ผลลัพธ์

## สรุปข้อเสนอ

ค่า default ควรใช้เพื่อช่วยให้เกษตรกรเริ่มประเมินได้ง่าย ไม่ใช่เพื่อแทนค่าจริงทั้งหมด การประเมินที่เหมาะสมควรใช้ภาพหรือวิดีโอเพื่อประเมิน posture ผ่าน MoveNet และใช้ค่า default เฉพาะข้อมูลที่ระบบมองไม่เห็นจากภาพ เช่น น้ำหนัก ระยะทาง ความถี่ ระยะเวลา และแรงดัน/ลาก จากนั้นให้ผู้ใช้หรือเจ้าหน้าที่วิจัยยืนยันข้อมูลก่อน export หรือใช้เป็นข้อมูลวิจัยต่อไป
